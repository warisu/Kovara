# Kovara Mini App Communication Protocol

This document specifies the host ↔ mini-app postMessage communication protocol.

## Overview

Kovara mini-apps run inside a sandboxed `iframe` or `webview`. All communication with the Kovara host client (which manages wallets, keys, and user profiles) happens asynchronously via the browser's standard `window.postMessage` API.

## Message Envelope

Every message sent between the host and a mini-app must conform to the following JSON structure:

```json
{
  "target": "Kovara",
  "id": "unique-message-id",
  "method": "method_name",
  "params": {},
  "error": null,
  "result": {}
}
```

### Request Envelope

- `target`: Must be the literal string `"Kovara"`.
- `id`: A unique string generated by the sender to match responses to requests.
- `method`: The name of the API method to call.
- `params`: An object containing the arguments for the API method.

### Response Envelope

- `target`: Must be the literal string `"Kovara"`.
- `id`: Matches the `id` of the original request.
- `result`: The return value of the call (if successful).
- `error`: An error object containing `code` and `message` if the call failed.

---

## Permission Matrix

Mini-apps must declare their required permissions in their manifest. The host validates requests against these permissions.

| Permission     | Description                                                               |
| :------------- | :------------------------------------------------------------------------ |
| `wallet.read`  | Allows reading the user's public Stellar address.                         |
| `wallet.sign`  | Requests the user to sign a Stellar transaction payload.                  |
| `profile.read` | Allows reading the user's Kovara profile details (e.g. username, avatar). |
| `post.create`  | Allows publishing a new post to the user's feed.                          |

---

## API Methods

### 1. `wallet.read`

Returns the active Stellar public key.

- **Request:**
  ```json
  {
    "target": "Kovara",
    "id": "1",
    "method": "wallet.read",
    "params": {}
  }
  ```
- **Response:**
  ```json
  {
    "target": "Kovara",
    "id": "1",
    "result": {
      "address": "GBBDQJ..."
    }
  }
  ```

### 2. `wallet.sign`

Prompts the user to sign an XDR transaction envelope.

- **Request:**
  ```json
  {
    "target": "Kovara",
    "id": "2",
    "method": "wallet.sign",
    "params": {
      "xdr": "AAAAAgAAAAD..."
    }
  }
  ```
- **Response:**
  ```json
  {
    "target": "Kovara",
    "id": "2",
    "result": {
      "signedXdr": "AAAAAgAAAAD..."
    }
  }
  ```

### 3. `profile.read`

Retrieves the user's Kovara profile details.

- **Request:**
  ```json
  {
    "target": "Kovara",
    "id": "3",
    "method": "profile.read",
    "params": {}
  }
  ```
- **Response:**
  ```json
  {
    "target": "Kovara",
    "id": "3",
    "result": {
      "username": "stellar_dev",
      "displayName": "Stellar Dev",
      "avatarUrl": "https://..."
    }
  }
  ```

### 4. `post.create`

Prompts the user to post content.

- **Request:**
  ```json
  {
    "target": "Kovara",
    "id": "4",
    "method": "post.create",
    "params": {
      "content": "Hello from a Kovara Mini App!"
    }
  }
  ```
- **Response:**
  ```json
  {
    "target": "Kovara",
    "id": "4",
    "result": {
      "success": true,
      "postId": "987"
    }
  }
  ```
